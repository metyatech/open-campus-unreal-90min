<Prerequisites>
- `OpenCampusUE90min` プロジェクトが開いていること
</Prerequisites>

ここから実際に手を動かして、コインを集めてポータルに入ってゴール、というゲームを完成させます。まずテストプレイの方法を確認し、そのあとコイン数表示とクリア条件を作っていきます。困ったら手を挙げてください。

<Section title="Step 1：テストプレイする（15〜25分）" goal="エディターでテストプレイできるようになります">

<Concept title="ビューポートとは">
Unreal Engine の画面中央にある大きな 3D ビューエリアです。ゲームのステージをリアルタイムに確認・編集する作業台として使います。
</Concept>

<Action img="./img/play-in-editor.png" alt="Unreal Editor のツールバーで、① プレイ ボタン、② 入力を受け付けるビューポートが示されているスクリーンショット">
  ①「**▶ プレイ**」をクリックしたあと、② **ビューポートをクリック**してゲーム操作を有効にします（クリックでカーソルが消えます）。
</Action>

<Recovery title="プレイから抜けられなくなったとき">
画面にゲームが映ったまま操作を受け付けない場合は、**Esc キー**を押すとテストプレイを終了できます。
</Recovery>

<Action>
  次のキーでキャラクターを動かしてみましょう。

  | キー | 動作 |
  | --- | --- |
  | **W / A / S / D** | 前後左右に移動 |
  | **Space** | ジャンプ |
  | **マウスの移動** | カメラの向きを変える |

</Action>

<Recovery title="キャラクターが動かないとき">
キーを押しても動かない場合は、ビューポートに入力が届いていません。ビューポートを 1 回クリックしてから、もう一度 W/A/S/D を押してください。
</Recovery>

<Action>
  確認できたら **Esc を押してテストプレイを終了**します。
</Action>

<Verify>W/A/S/D でキャラクターが動き、Space でジャンプし、Esc で編集画面に戻れれば成功です。</Verify>

<Checkpoint>
- W/A/S/D でキャラクターが動く
- Space でジャンプできる
- Esc でテストプレイを終了できる
</Checkpoint>

</Section>

<Section title="Step 2：BP_Coin を 10 枚以上置く（25〜40分）" goal="レベルに拾えるコインを 10 枚以上配置します">

<Concept title="BP_Coin とは">
`BP_Coin` は、触れると拾えるコインとして用意されている Blueprint クラスです。今日はこのコインを新しく作らず、すでにある `BP_Coin` をレベルに置いて使います。あとで `BP_Coin` の中を少し直すと、拾った枚数が画面に増えるようになります。
</Concept>

<Action>
  この後の作業では、ビューポートの視点を動かしながら `BP_Coin` を置きます。ビューポート内で、次の操作を試してみましょう。

  | 操作 | 動作 |
  | --- | --- |
  | **右クリック + ドラッグ** | 見る向きを変える |
  | **右クリック中に W / A / S / D** | 前後左右に移動 |
  | **右クリック中に Q / E** | 下・上に移動 |
  | **右クリック中にマウスホイール** | 移動速度を変える（上で速く、下でゆっくり） |

  プレイ中のキャラクター操作と違い、編集画面で視点を動かすときは**右クリックを押し続ける**必要があります。

</Action>

<Section title="2-1. コインを置いて場所を直す" goal="レベルにコインを 10 枚以上配置します">

<Action img="./img/bp-coin-drag-from-content-browser.png" alt="Unreal Editor で、① Content Drawer、② OpenCampus フォルダ、③ BP_Coin、④ 配置先のビューポートが示されているスクリーンショット">
  ① Content Drawer で ② `OpenCampus` フォルダを開き、③ `BP_Coin` を ④ ビューポートへドラッグ＆ドロップして配置します。
</Action>

<Action img="./img/how-to-translate-with-translation-gizmo.png" alt="ビューポート上で選択されたオブジェクトに、移動ギズモの矢印が表示されているスクリーンショット">
  移動ギズモの矢印をドラッグして、歩いて取れる高さや場所へ動かします。
</Action>

<Recovery title="矢印（移動ギズモ）が表示されないとき">
矢印が出ない場合は、`BP_Coin` が選択されていないか、移動モードになっていない可能性があります。ビューポートまたはアウトライナーで `BP_Coin` をクリックし、それでも出なければ **W キー** を押すか、![平行移動に切り替えボタン](./img/translate-button.png) をクリックして移動モードに切り替えてください。
</Recovery>

<Action img="./img/bp-coin-placed-in-level.png" alt="① ビューポートに配置された複数のコインと、② アウトライナーに並ぶ BP_Coin が示されているスクリーンショット">
  同じ操作をくり返し、① レベル内に `BP_Coin` を 10 枚以上置きます。② アウトライナーでも枚数を確認できます。歩くだけで取れる場所を多めにし、余裕があればジャンプで取る場所も少し入れます。
</Action>

<Verify>ビューポートまたはアウトライナーで、`BP_Coin` が 10 枚以上あることを確認できれば配置完了です。</Verify>

</Section>

:::tip[こまめに保存]
作業中は、**Ctrl + Shift + S** ですべての修正をまとめて保存できます。保存していない変更は、トラブル時に失われることがあるため、修正したらこまめに保存しましょう。
:::

<Section title="2-2. 動作確認をする" goal="コインは拾えるがコイン枚数の表示は増えないことを確認します">

<Action img="./img/playtest-coin-counter-zero.png" alt="プレイ中のビューポートで、① 拾うコインと、② 0枚のままのコイン数表示が示されているスクリーンショット">
  プレイして、① 置いた `BP_Coin` を 1〜2 枚拾います。② 表示が `0枚` のまま変わらないことを確認します。
</Action>

<Action>
  確認できたら Esc でテストプレイを終了します。
</Action>

<Verify>コインは拾えますが、画面の表示が `0枚` のまま変わらない状態です。</Verify>

</Section>

<Checkpoint>
- `BP_Coin` が 10 枚以上置かれている
- コインを拾える
- 画面の表示はまだ `0枚` のままになっている
</Checkpoint>

</Section>

<Section title="Step 3：BP_Coin を拾ったときに枚数を更新する（40〜55分）" goal="コインを拾うたびに画面の枚数表示が増えるようになります">

<Section title="3-1. BP_Coin を開く" goal="コインの Blueprint を編集できるようになります">

<Action img="./img/open-bp-coin-from-content-browser.png" alt="コンテンツブラウザで、① Content Drawer、② OpenCampus フォルダ、③ BP_Coin をダブルクリックして開く操作が示されているスクリーンショット">
  ① Content Drawer で ② `OpenCampus` フォルダを開き、③ `BP_Coin` をダブルクリックして開きます。
</Action>

<Verify>Blueprint エディターが開き、`BP_Coin` の編集画面が表示されれば準備完了です。</Verify>

<Concept title="イベントグラフとは">

イベントグラフは、Blueprint の中でゲームの動きを作る場所です。
ノードを配置して、**触れたらどう動くか**を組み立てていきます。最初からある `Event ActorBeginOverlap` が「何かに触れた瞬間」のスタート地点です。

</Concept>

<Action img="./img/BP_Item-EventGraph.png" alt="イベントグラフの場所を表すスクリーンショット">
  イベントグラフ内を自由に動かして、操作に慣れましょう。

| 操作 | 動作 |
| --- | --- |
| **右クリック + ドラッグ** | グラフをスクロール（パン） |
| **マウスホイール** | ズームイン／アウト |
| **Home キー** | 全ノードが画面に収まるように表示を合わせる |

</Action>

<Action img="./img/bp-coin-yellow-comment-add-one-empty.png" alt="BP_Coin のイベントグラフで、黄色のコメントエリア「コインを1枚増やす」が示されているスクリーンショット">
  イベントグラフを操作して、黄色のコメントエリア「**コインを1枚増やす**」を見つけましょう。ここが、これから処理を追加する場所です。
</Action>

<Verify>`BP_Coin` で黄色いコメントエリア「**コインを1枚増やす**」を見つけられれば準備完了です。</Verify>

</Section>

<Section title="3-2. コイン数を 1 増やす" goal="BP_Coin が拾われたときにコイン数を 1 増やします">

<Action img="./img/bp-coin-add-get-game-instance.png" alt="BP_Coin のイベントグラフで、① 黄色いコメントエリアの空白部分、② 検索欄、③ Get Game Instance の候補が示されているスクリーンショット">
  ① 黄色いコメントエリア **コインを1枚増やす** の中の空いている場所を右クリックします。② 表示された検索欄に `Get Game Instance` と入力し、③ 候補から `Get Game Instance` を選びます。
</Action>

<Action img="./img/bp-coin-add-update-coins.png" alt="BP_Coin のイベントグラフで、① Get Game Instance の出力ピン、② 検索欄、③ Update Coins の候補が示されているスクリーンショット">
  ① `Get Game Instance` の出力ピンからドラッグし、② 空いている場所へドラップします。② 表示された検索欄に `Update Coins` と入力し、③ 候補から `Update Coins` を選びます。
</Action>

<Action img="./img/bp-coin-move-update-coins.png" alt="BP_Coin のイベントグラフで、Update Coins ノードをドラッグして、Pawn に当たった場合のみとビジュアルの更新の間に移動する操作が示されているスクリーンショット">
  `Update Coins` を「**Pawn に当たった場合のみ**」と「**ビジュアルの更新**」の2つのノードの間に位置するように、画像の辺りの位置へ移動します。
</Action>

<Action img="./img/bp-coin-connect-update-coins.png" alt="BP_Coin のイベントグラフで、Update Coins ノードの左右の白い実行ピンをそれぞれ繋ぐ操作が示されているスクリーンショット">
  `Update Coins` の左右の白い実行ピンを、それぞれ繋ぎます。左は「**Pawn に当たった場合のみ**」の実行ピンから、右は「**ビジュアルの更新**」の実行ピンへつなぎます。
</Action>

<Action img="./img/bp-coin-set-update-coins-change-one.png" alt="BP_Coin のイベントグラフで、Update Coins ノードの Change に 1 を入力する場所が示されているスクリーンショット">
  `Update Coins` の `Change` を `1` にします。
</Action>

<Concept title="「コンパイル」とは">

Blueprint のノードを組み終わったら、**コンパイル**でゲームに反映します。文書を「保存」するのに近い操作ですが、正確には「作ったノードの設定をゲームが実際に動かせる形に変換する」作業です。コンパイルしないと、ノードを変更してもゲームには反映されません。

ツールバーのコンパイルボタンのアイコンで結果がわかります。

| アイコン | 意味 |
| --- | --- |
| **緑のチェック** | 成功。ゲームに反映されました |
| **赤の ×** | エラーあり。修正が必要です |

ノードを組み終わったら一度押してアイコンを確認する習慣をつけておくと、問題を早く発見できます。

</Concept>

<Action img="./img/compile-BP_Item-and-check-result.png" alt="Blueprint エディタのツールバー。コンパイルボタンがハイライトされている">
  「**コンパイル**」をクリックします。
</Action>

<Recovery title="コンパイルアイコンが赤い × のとき">
コンパイルエラーです。画面下部の「出力ログ」パネルにエラーメッセージが表示されます。エラー行をダブルクリックすると問題のノードにジャンプできます。修正したら、もう一度「コンパイル」を押してください。
</Recovery>

<Verify img="./img/compile-success.png" alt="コンパイル成功。アイコンが緑のチェックマークになっている">コンパイルアイコンがこのような**緑のチェック**になれば設定完了です。</Verify>

</Section>

<Section title="Step 4：BP_Portal のクリア条件を直す（55〜70分）" goal="10 枚以上集めてポータルに入るとゲームをクリアできるようになります">

<Concept title=">= 10 と Branch">
`>= 10` は、数が 10 以上かどうかを調べる条件です。`Branch` は、条件が合っているときと合っていないときで処理の行き先を分けます。ここでは、コインが 10 枚以上ならポータルを有効にするために使います。
</Concept>

<Section title="4-1. BP_Portal を開く" goal="ポータルの Blueprint を編集できるようになります">

<Action img="./img/open-bp-portal-from-content-browser.png" alt="① Content Drawer で ② `OpenCampus` フォルダを開き、③ `BP_Portal` をダブルクリックして開く操作が示されているスクリーンショット">
  ① Content Drawer で ② `OpenCampus` フォルダを開き、③ `BP_Portal` をダブルクリックして開きます。
</Action>

<Action img="./img/bp-portal-yellow-comment-condition-empty.png" alt="BP_Portal のイベントグラフで、① ポータルの判定処理、② 黄色のコメントエリア「集めたコインが10枚以上なら」が示されているスクリーンショット">
  イベントグラフを操作し、黄色のコメントエリア「**集めたコインが10枚以上なら**」を見つけます。ここに、これから条件の処理を追加していきます。
</Action>

<Verify>`BP_Portal` で黄色いコメントエリア「**集めたコインが10枚以上なら**」を見つけられれば準備完了です。</Verify>

</Section>

<Section title="4-2. 10 枚以上で通れる条件にする" goal="BP_Portal がコイン 10 枚以上のときだけクリア処理へ進むようになります">

<Action img="./img/bp-portal-condition-greater-equal-10-wired.png" alt="BP_Portal のイベントグラフで、① >= 10 の比較ノード、② Branch の条件への接続が示されているスクリーンショット">
  ① `>= 10` の比較ノードを置き、② 比較結果を `Branch` の `Condition` につなぎます。
</Action>

<Action img="./img/compile-success.png" alt="Blueprint エディター上部で、① コンパイル、② 保存の操作場所が示されているスクリーンショット">

</Action>

<Recovery title="ポータルが反応しないとき">
10 枚以上集めても反応しない場合は、`>= 10` の結果が `Branch` につながっていない可能性があります。つながりを確認してから、もう一度コンパイルしてください。
</Recovery>

<Verify>コンパイルアイコンが**緑のチェック**になれば設定完了です。</Verify>

</Section>

<Section title="Step 5：クリアに必要なコイン数を調整する（70〜80分）" goal="BP_Portal の数字を変えて、クリア条件を自分で調整できるようになります">

<Concept title="しきい値とは">
しきい値は、「この数以上なら OK」と決める境目の数字です。`>= 10` の `10` を変えると、何枚集めたら `BP_Portal` が反応するかを調整できます。ゲームの難しさを変えるときは、見た目だけでなく、このような条件の数字も調整します。
</Concept>

<Section title="5-1. 3 枚でクリアできるようにする" goal="少ないコイン数でもクリアできる条件に変えます">

<Action img="./img/bp-portal-condition-greater-equal-3.png" alt="BP_Portal のイベントグラフで、① 比較ノード、② しきい値 3 が示されているスクリーンショット">
  ① 比較ノードを確認し、② 条件を `>= 3` に変更してから、コンパイルして保存します。
</Action>

<Action img="./img/playtest-clear-with-3-coins.png" alt="プレイ中のビューポートで、① 3枚になったコイン数表示、② 光った BP_Portal が示されているスクリーンショット">
  プレイして、① `BP_Coin` を 3 枚拾ってから、② `BP_Portal` に入ります。
</Action>

<Verify>3 枚でゲームクリアになれば、条件の数字で難しさを変えられることを確認できています。</Verify>

</Section>

<Concept title="M_Coin とは">
`M_Coin` は、コインの見た目を決めるマテリアルです。Blueprint がゲームの動きを作るのに対して、マテリアルは色や動き方などの見た目を調整します。
</Concept>

ここでは Blueprint ではなく、`M_Coin` のマテリアルで見た目を少しアレンジします。

### 演習-発展1：M_Coin を時間で回す

<Exercise>

![M_Coin のマテリアルグラフで、① 黄色のコメントエリア「時間経過でコインを回す」と、② 接続先の A ピンが示されているスクリーンショット](./img/m-coin-yellow-comment-rotate-empty.png)

`M_Coin` を開き、① 黄色いコメントエリア **時間経過でコインを回す** の中で、② `Time` ノードを `A` ピンにつないでください。

確認方法：保存してプレイし、コインの見た目が時間に合わせて回っていれば成功です。

<Solution>

![M_Coin のマテリアルグラフで、① Time ノードの出力と、② A ピンへの接続が示されているスクリーンショット](./img/m-coin-time-to-a-wired.png)

① `Time` ノードの出力が、② `A` ピンにつながっていれば正解です。

</Solution>
