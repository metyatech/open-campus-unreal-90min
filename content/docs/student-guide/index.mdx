<Prerequisites>

ここから実際に手を動かしてゲームを作っていきます。

このページの手順をひとつずつ進めれば、最後にはキャラクターが動いてアイテムを集めてクリアになるゲームが完成します。何か困ったら、すぐにサポートしに行きますので、お気軽に手を挙げて教えてください。

<Section title="1-3. プレイして動かしてみる" goal="ゲームをテストプレイできるようになります">

<Concept title="ビューポートとは">
Unreal Engine の画面中央にある大きな 3D ビューエリアです。ゲームのステージをリアルタイムに確認・編集する作業台として使います。
</Concept>

<Action img="./img/play-in-editor.png" alt="Unreal Editor ツールバー。①プレイボタンと②ビューポートがハイライトされている">
  ① 「**▶ プレイ**」をクリックしたあと、② **ビューポートをクリック**してゲーム操作を有効にします（クリックでカーソルが消えます）
</Action>

<Recovery title="プレイから抜けられなくなったとき">
画面にゲームが映ったまま操作を受け付けない場合は、**Esc キー**を押すとテストプレイを終了できます。
</Recovery>

<Action>
  次のキーでキャラクターを動かしてみましょう

  | キー | 動作 |
  | --- | --- |
  | **W / A / S / D** | 前後左右に移動 |
  | **Space** | ジャンプ |
  | **マウスの移動** | カメラの向きを変える |

</Action>

<Recovery title="操作できないとき">
キーを押してもキャラクターが動かない場合、ビューポートがフォーカスされていません。ビューポートを 1 回クリックすると、キー入力がゲームに届くようになります。
</Recovery>

<Action>
  確認できたら **Esc を押してプレイを終了**します
</Action>

<Verify>ゲームをプレイできて、W/A/S/D でキャラクターが動き、Space でジャンプできれば成功です。</Verify>

</Section>

<Checkpoint>

- W/A/S/D でキャラクターが動く
- Space でジャンプできる
- Esc でプレイを終了できる

</Checkpoint>

</Section>

  | 操作 | 動作 |

<Section title="2-3. キューブを Blueprint クラスに変換する" goal="キューブが BP_Item という Blueprint クラスになります">

<Concept title="Blueprint クラスとは">

Unreal Engine でオブジェクトに「触れたら消える」「スコアが増える」といった動作を設定するには、**Blueprint クラス**にする必要があります。

Word や Google スライドの「テンプレート」を想像してください。テンプレートを 1 つ作れば、同じデザインのコピーを何個でも作れます。Blueprint クラスも同じで、1 個設定すれば 5 個に増やせます。

</Concept>

<Concept title="「BP_」プレフィックスの意味">

この後のダイアログでは、Blueprint の名前を **BP_Item** にします。ファイル名の先頭に「BP_」を付けるのは、一覧を見たときに「これは **Blueprint** だ」とすぐわかるようにするための慣習です。

</Concept>

<Concept title="詳細パネルとは">
ビューポートなどで選択中のオブジェクトの設定項目（プロパティ）が一覧表示されるパネルです。通常は画面右側にあり、ここで選んだものの見た目や動作を細かく変更していきます。
</Concept>

<Action img="./img/convert-to-blueprint-process.png" alt="ビューポートでキューブを選択し、詳細パネルにブループリントクラスへの変換ボタンが表示されている">
  ビューポートでキューブを**クリックして選択**し、詳細パネルの「**ブループリントクラスへの変換**」ボタンを押します
</Action>

<Action img="./img/convert-to-blueprint-window-process.png" alt="Blueprint クラス作成ダイアログ。①親クラス ②パス ③ブループリント名入力欄 ④アセットパス ⑤選択ボタン">
  ①②④ は画像の通りになっているか見比べて、違っていれば画像と同じ状態にします（最初から画像の通りになっているはずです）。③ のブループリント名に「**BP_Item**」と入力して、⑤「**選択**」をクリックします
</Action>

<Verify img="./img/after-convert-to-blueprint.png" alt="コンテンツブラウザに BP_Item が追加され、Blueprint 編集画面が自動で開いた状態">コンテンツブラウザに `BP_Item` が表示されて Blueprint 編集画面が自動で開けば変換成功です</Verify>

<Action img="./img/close-BP_Item-editor-window.png" alt="BP_Item 編集画面のタブ。閉じるボタンが示されている">
  自動的に開いた `BP_Item` の編集画面は今は使わないので、タブを閉じます
</Action>

:::tip[こまめに保存]
作業中は **Ctrl + Shift + S**（すべて保存）を使ってこまめに保存します。保存しないと、万が一のトラブルで作業が消えてしまうことがあります。
:::

</Section>

<Section title="演習：アイテムをどこに置くか計画しよう" goal="BP_Item を置きたい場所を 5 か所リストアップします">

<Exercise>

次のステップで BP_Item を 5 個配置します。その前に、**アイテムを置きたい場所を 5 か所**決めましょう。

プレイ（▶）してキャラクターを動かしながら、次の条件に合う場所を探します。

- 走ってすぐ取れる「楽な場所」を 2〜3 か所
- ジャンプが必要な「少し難しい場所」を 1〜2 か所
- 実際に足で立てること（壁の中・床の下は NG）

5 か所決まったら Esc でプレイを終了してください。

<Solution>

置き場所の例：

- 楽な場所：スタート地点の近く、平らな床の上
- 難しい場所：ステージ中央の台の上、端の高台

ゲームとしておもしろいのは、**どれも取れるけど全部が同じ難易度ではない**配置です。次のステップでこの計画を実現していきましょう。

</Solution>
</Exercise>

</Section>

<Section title="Step 3：触れたら消えてスコアが増えるようにする（30〜55分）" goal="アイテムに触れるたびにアイテムが消えてスコアが増えるようになります">

どちらか片方だけ設定してもゲームとして成り立ちません。両方を一緒に作ります。

| 仕組み | 片方だけ設定するとどうなる？ |
| --- | --- |
| **アイテムが消える** | スコアは増えない |
| **スコアが 1 増える** | アイテムが消えない（何度でも取れてしまう） |

<Section title="3-1. プレイヤーキャラクターに Score 変数を追加する" goal="BP_ThirdPersonCharacter に Integer 型の Score 変数を追加します">

スコアを記録する「変数」を作ります。

<Concept title="変数はどこに作る？">

変数はどの Blueprint にでも作れます。ただし、その変数を「誰が持つのか」を考えて置き場所を決める必要があります。

スコアは「このプレイヤーが何個取ったか」という情報なので、**プレイヤーキャラクターの Blueprint（BP_ThirdPersonCharacter）** に作るのが自然です。プレイヤーはゲーム中は常に存在するので、途中で消えてスコアが消える心配もありません。

</Concept>

<Concept title="変数の型について">

変数は「何かを入れておける入れ物」ですが、入れ物ごとに **入れられるものの種類（型）** が決まっています。

| 型 | 入れられるもの | 使いどころ |
| --- | --- | --- |
| **Integer（整数）** | 1, 2, 3 … | スコア、個数、残機など |
| **Boolean（ブーリアン）** | True / False | フラグ、オン/オフの状態など |
| **String（文字列）** | "Hello" … | 名前、メッセージなど |

今回のスコアは「1個取った」「2個取った」のように**整数で十分**なので、Integer を選びます。

</Concept>

<Action img="./img/how-to-open-BP_ThirdPersonCharacter.png" alt="コンテンツブラウザ。①フォルダ階層と ②BP_ThirdPersonCharacter が示されている">
  ①→② の順で **BP_ThirdPersonCharacter** をダブルクリックで開きます
</Action>

<Action img="./img/add-Score-variable.png" alt="Blueprint 変数パネル。①追加ボタン ②変数名 Score ③型 Integer が示されている">
  ①「**＋**」から、② 名前「**Score**」・③ 型「**Integer**」の変数を追加します
</Action>

<Verify>変数パネルに Integer 型の `Score` が追加されていれば設定完了です</Verify>

<Action>
  タブの「**×**」をクリックして BP_ThirdPersonCharacter の編集画面を閉じます
</Action>

</Section>

<Section title="3-2. BP_Item のコリジョン設定を変える" goal="プレイヤーが BP_Item をすり抜けて通れるようになります">

<Concept title="コリジョンとは">

**コリジョン（Collision）** とは、オブジェクト同士が「ぶつかった・触れた」ことをゲームが認識するための当たり判定のことです。

| 種類 | 動作 | 使いどころ |
| --- | --- | --- |
| **ブロック** | 相手をはじいて通れなくする | 壁・床・障害物 |
| **オーバーラップ** | すり抜けられる＋触れたことだけ通知する | アイテム・ゴールゾーン |

BP_Item は元キューブなので初期設定は「ブロック」です。「オーバーラップ」に変えることで、すり抜けつつ触れた瞬間を検知できるようになります。

</Concept>

<Action>
  コンテンツブラウザで「**BP_Item**」をダブルクリックして編集画面を開きます
</Action>

<Action img="./img/how-to-BP_Item_OverlapAllDynamic.png" alt="BP_Item の詳細パネル。①コリジョン設定 ②Generate Overlap Events ③コリジョンプリセット OverlapAllDynamic が示されている">
  ①「**コリジョン**」で検索して、②「**Generate Overlap Events**」をオン、③「**コリジョンプリセット**」を ④「**OverlapAllDynamic**」にします
</Action>

<Verify img="./img/BP_Item_GenerateOverlapEvents_and_OverlapAllDynamic.png" alt="BP_Item の詳細パネル。Generate Overlap Events がオン、コリジョンプリセットが OverlapAllDynamic に切り替わっている">**Generate Overlap Events** がオン、**コリジョンプリセット** が **OverlapAllDynamic** に切り替わっていれば設定完了です</Verify>

<Action>
  メインエディタに戻ってプレイ（▶）して、キャラクターをアイテムに向かって歩かせます
</Action>

<Verify>キャラクターがアイテムをすり抜けて通れれば成功です。確認できたら Esc でプレイを終了します。</Verify>

</Section>

<Section title="3-3. Overlap イベントで動作確認する" goal="アイテムに触れると画面に「Hello」が表示されるようになります">

アイテムに触れたとき、画面に「Hello」という文字が表示されるようにします。まずこれで「**触れたことを検知する**」仕組みの作り方を学びます。

<Action img="./img/open-full-blueprint-editor-button.png" alt="BP_Item 編集画面。フルブループリントエディタを開くボタンがハイライトされている">
  `BP_Item` の編集画面の、「**フルブループリントエディタを開く**」ボタンをクリックします
</Action>

<Concept title="イベントグラフとは">

![BP_Item のイベントグラフ](./img/BP_Item-EventGraph.png)

黄色枠の部分がイベントグラフです。ここにノードを配置したりつなげたりして**ゲームの仕組みを書いていきます**。

</Concept>

<Action>
  次の操作でイベントグラフを自由に動かせます。作業を始める前に試してみましょう。

  | キー / 操作 | 動作 |
  | --- | --- |
  | **右クリック + ドラッグ** | グラフをスクロール（パン） |
  | **マウスホイール** | ズームイン／アウト |
  | **Home キー** | 全ノードが画面に収まるように表示を合わせる |

</Action>

<Concept title="Event ActorBeginOverlap と Print String">

- **Event ActorBeginOverlap**：「何かが触れた瞬間」に起動するノードです。
- **Print String**：指定した文字を画面に表示するノードで、仕組みが正しく動いているかを手軽に確認するために使います。

</Concept>

<Action img="./img/node-Event%20ActorBeginOverlap.png" alt="イベントグラフ上の Event ActorBeginOverlap ノード">
  グラフ内で `Event ActorBeginOverlap` と書かれたノードを探します
</Action>

<Recovery title="グラフ内でノードが見当たらないとき">
ビューが動いてノードが画面外に出ている可能性があります。**Home キー**を押すと全ノードが収まる表示に戻るので、そこから探し直してください。
</Recovery>

<Action img="./img/add-print-string-node-from-actorbeginoverlap.png" alt="Event ActorBeginOverlap の ①実行ピンからドラッグし、②空白で離して ③Print String を検索、④選択する手順">
  ① **実行ピン（白い矢印）** からドラッグして、② グラフ上の何もないところで離し、③「**Print String**」を検索して、④ が選択されている状態で **Enter** で追加します
</Action>

<Recovery title="ノードが見つからないとき">
検索欄に正しい名前が入力されているか確認してください。スペルが少しでも違うと表示されません。
</Recovery>

<Verify img="./img/BP_Item-print-string-from-actorbeginoverlap.png" alt="Event ActorBeginOverlap から Print String に接続されたイベントグラフ">イベントグラフがこの形になっていればOKです</Verify>

<Concept title="「コンパイル」とは">

文書を「保存」するのに近い操作です。**作ったノードの設定をゲームが実際に動かせる形に変換する作業**で、コンパイルしないとゲームに反映されません。アイコンが緑のチェック → 成功、赤の × → エラーあり。ノードを組み終わったら一度押してアイコンを確認する習慣をつけておくと、問題を早く発見できます。

</Concept>

<Action img="./img/compile-BP_Item-and-check-result.png" alt="Blueprint エディタのツールバー。コンパイルボタンがハイライトされている">
  「**コンパイル**」をクリックします
</Action>

<Recovery title="コンパイルアイコンが赤い × のとき">
コンパイルエラーです。画面下部の「出力ログ」パネルにエラーメッセージが表示されます。エラー行をダブルクリックすると問題のノードにジャンプできます。エラーを修正したら、もう一度「コンパイル」を押してください。
</Recovery>

<Verify img="./img/compile-success.png" alt="コンパイル成功。アイコンが緑のチェックマークになっている">問題なければ、コンパイルアイコンが**緑のチェック**に変わります</Verify>

<Action>
  メインエディタに戻ってプレイ（▶）して、キャラクターをアイテムに触れさせます
</Action>

<Verify img="./img/print-string-Hello-in-viewport.png" alt="プレイ中のビューポート。画面左上に水色で Hello と表示されている">画面左上に水色で「Hello」と表示されれば成功です</Verify>

<Action>
  `Print String` ノードをクリックして選択し、**Delete キー**を押して削除します
</Action>

<Action>
  「**コンパイル**」をクリックします
</Action>

<Concept title="Branch（条件分岐）とは">

日常でも「もし雨なら傘を持つ、晴れなら持たない」のように条件で行動を変えます。Blueprint の **Branch** も同じで、**Condition（条件）** が `True` なら **True** 側の処理へ、`False` なら **False** 側の処理へ進みます。

今回は「**Score が 5 以上か？**」を Condition にして、`True` のときだけ `CLEAR!` を表示します。比較には「**>=**（以上）」ノードを使い、**Score** と **5** を比べて `True` / `False` を返してもらいます。

</Concept>

<Action img="./img/connect-is-score-clear-to-branch-condition.png" alt=">= ノードの出力が Branch の Condition ピンに接続された状態">

## 振り返り

今日作ったゲームには、3つのプログラミングの基本的な仕組みが使われています。

| 仕組み | 意味 |
| --- | --- |
| **イベント** | 「何かに触れたとき」に処理を開始する |
| **変数** | スコアをゲームの中で記憶する |
| **条件分岐** | 「スコアが足りているかどうか」で処理を分ける |

この3つはどんなゲームにも、ゲーム以外のアプリにも共通して出てきます。

### 演習-発展3：取ったときにキラキラエフェクトを出す

<Exercise>

アイテムに触れた瞬間に、派手な粒子エフェクトが出るようにしましょう。

**やり方 1（Niagara システムを作る）**

1. コンテンツブラウザの空いている場所で**右クリック** →「**FX**」→「**Niagara System**」

4. ノードの「**System Template**」ドロップダウンで「**NS_Collect**」を選ぶ
5. 「**Location**」ピンからドラッグ → 空の場所で離し →「**Get Actor Location**」を検索して追加

### 演習-発展5：スコアを画面に表示する

<Exercise>

現在のスコアを画面にリアルタイムで表示する **HUD** を追加します。未習の機能が多いので、手順に沿って進めてください。

**やり方 1（HUD の Widget を作る）**

1. コンテンツブラウザで**右クリック** →「**ユーザーインターフェース**」→「**Widget Blueprint**」→「**User Widget**」
2. 名前を「**WBP_HUD**」にしてダブルクリックで開く
3. 左上の **パレット** から「**Text**」を、中央の **階層** パネルの `Canvas Panel` の上にドラッグ
4. 追加した `Text` を選び、詳細パネルの「**Content**」→「**Text**」欄の右にある「**バインド**」→「**バインディングを作成**」をクリック
5. 関数グラフが開きます。空の場所で**右クリック** →「**Get Player Character**」を検索して追加
6. `Get Player Character` の**出力ピン**（青）からドラッグ → 空の場所で離し →「**Cast To BP_ThirdPersonCharacter**」を検索して追加
7. Cast の「**As BP Third Person Character**」ピン（青）からドラッグ → 空の場所で離し →「**Get Score**」を検索して追加
8. `Get Score` の出力ピン（緑）を、右端の **Return Node** の「**Return Value**」ピン（紫）にドラッグしてつなぐ（Integer → Text の変換ノードが自動で挿入されます）
9. コンパイル・保存

**やり方 2（ゲーム開始時に HUD を画面に出す）**

1. `BP_ThirdPersonCharacter` を開く
2. イベントグラフに `Event BeginPlay` ノードが無ければ、右クリック →「**Event BeginPlay**」で追加
3. `BeginPlay` の実行ピンに「**Create Widget**」ノードをつなぎ、**Class** に `WBP_HUD` を指定
4. `Create Widget` の **Return Value** から「**Add to Viewport**」を追加してつなぐ
5. コンパイル・保存してプレイ

→ 画面にスコア（最初は `0`）が表示され、アイテムを取るたびに数字が増えれば成功です。
