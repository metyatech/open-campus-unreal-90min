このページは、授業中にそのまま見ながら進める手順書です。  
各ステップで「目標」「作業」「チェック」を順番に進めてください。

## 0. はじめに（0〜10分）

- 目標：プロジェクトを作成し、キャラクターが動く状態にする
- 作業：
  1. Unreal Engine 5.x を起動する
  2. `Games` → `Third Person` を選ぶ
  3. `Blueprint` を選び、プロジェクトを作成する
  4. 右上の `Play` で再生する
- チェック：
  - `W A S D` で移動できる
  - `Space` でジャンプできる
- つまずいたとき：
  - 再生できない：`Compile` ではなく、まず `Play` を押せているか確認する
  - 操作できない：再生開始後に画面をクリックして入力フォーカスを当てる

---

## 1. アイテムを作って配置する（10〜30分）

- 目標：回収対象となるアイテムを5個配置する
- 作業：
  1. `Place Actors` から `Cube` を1個置く
  2. 置いたCubeを選択し、`Convert Selected Actor to Blueprint Class` を実行する
  3. Blueprint名を `BP_Item` にする
  4. `BP_Item` をレベルに合計5個配置する
- チェック：
  - `Outliner` 上に `BP_Item` が5個見える
  - ただの `Cube` ではなく `BP_Item` を配置している
- つまずいたとき：
  - ただのCubeを複製していた：`BP_Item` を作り直し、既存のCubeは削除して置き換える
  - どれが本物かわからない：`Outliner` の名前で `BP_Item` を確認する

---

## 2. 触れたらアイテムが消える（30〜50分）

- 目標：プレイヤーが触れたアイテムだけ消えるようにする
- 作業（`BP_Item`）：
  1. `BP_Item` を開く
  2. `Box Collision` を追加し、アイテム全体を包むサイズにする
  3. Event Graphで `OnComponentBeginOverlap (Box)` を追加する
  4. `Destroy Actor` につなぐ
  5. `Compile` と `Save` を押してプレイ確認する
- チェック：
  - 触れたアイテムがその場で消える
  - 触れていないアイテムは残る
- つまずいたとき：
  - 反応しない：`Box Collision` の Overlap イベントを使っているか確認する
  - 消えない：ノードの線が `BeginOverlap -> Destroy Actor` でつながっているか確認する

---

## 3. スコアを増やす（50〜65分）

- 目標：アイテムを1個取るごとにスコアを1増やす
- 作業：
  1. `ThirdPersonCharacter` に `Score` 変数を作る（型は `Integer`、初期値 `0`）
  2. `BP_Item` の Overlap で、重なった相手がプレイヤーか判定する
  3. プレイヤーなら `Score` を `+1` してから `Destroy Actor` する
- チェック：
  - 2個取るとスコアが2になる
  - 3個取るとスコアが3になる
- つまずいたとき：
  - `Score` が増えない：先に `Destroy Actor` していないか確認（加算を先に実行）
  - 型が違う：`Score` が `Boolean` なら `Integer` で作り直す

---

## 4. ゴール判定を作る（65〜80分）

- 目標：必要数以上を回収してからゴールに入るとクリアできるようにする
- 作業（`BP_Goal`）：
  1. ゴール用Blueprint `BP_Goal` を作ってレベルに配置する
  2. `Box Collision` を追加する
  3. `RequiredScore`（`Integer`）を作り、値を `5` にする
  4. Overlapでプレイヤーの `Score` を取り、`Score >= RequiredScore` で分岐する
  5. 条件を満たしたらクリア処理（フラグONまたはクリア表示処理）につなぐ
- チェック：
  - 4個以下でゴールしてもクリアしない
  - 5個以上でゴールするとクリアになる
- つまずいたとき：
  - 5個ちょうどで失敗する：比較演算子が `>` になっていないか確認し、`>=` にする
  - ゴールできない：`Box Collision` の位置が高すぎないか確認する

---

## 5. クリア表示を出す（80〜88分）

- 目標：クリア時に結果が目でわかるようにする
- 作業：
  1. `Widget Blueprint` を1つ作る
  2. テキストに `CLEAR!` と入れる
  3. クリア条件を満たしたタイミングで `Create Widget -> Add to Viewport` を実行する
- チェック：
  - クリア時に `CLEAR!` が表示される

---

## 6. 最終確認（88〜90分）

- 目標：最初から最後までプレイできる状態で授業を終える
- 作業：
  1. 最初から通しでプレイする
  2. 次の3点を1分で説明できるようにする
     - 自分が作った処理
     - 詰まった箇所と直し方
     - 次に追加したい機能

---

## 困ったときの最短リカバリー

1. `Compile` と `Save` を押す
2. ノードの線が途中で切れていないか確認する
3. 変数の型（`Integer` / `Boolean`）を確認する
4. レベル上の配置を `Outliner` で確認する

直らない場合は、講師へ「どのステップで」「何をしたら」「どうなったか」の順で伝えてください。
