このページは、**授業中にそのまま見ながら進める手順書**です。

**今日のゴールは、キャラクターを動かし、アイテムを集め、条件を満たしたらクリアになるゲームを自分で完成させること**です。

## はじめに

- この授業は **90分の体験授業** です。全部を完璧に覚える必要はありません。
- 困ったときは、**今いるステップ名** と **何をしたらどうなったか** を講師に伝えてください。
- まずは見本どおりに作り、時間が余ったら自分なりの調整をします。

## 1. プロジェクトを作って遊べる状態にする（0〜15分）

**ここでの目標は、Unreal Engine を起動して、キャラクターが動く状態まで進むこと**です。

### やること

1. Unreal Engine 5.x を起動する
2. `Games` → `Third Person` を選ぶ
3. `Blueprint` を選び、新しいプロジェクトを作る
4. 右上の `Play` を押す

### うまくいっている目印

- `W A S D` で移動できる
- `Space` でジャンプできる

### 困ったとき

- 再生されない: `Play` を押せているか確認する
- 動かせない: 再生後に画面をクリックして入力を有効にする
- 画面が違う: 講師の画面と比べて、`Third Person` と `Blueprint` になっているか確認する

## 2. 取れるアイテムを作って置く（15〜35分）

**ここでは、「ゲームに何を集めるのか」を自分で作ります。**

### やること

1. `Place Actors` から `Cube` を1つ置く
2. その Cube を選び、`Convert Selected Actor to Blueprint Class` を実行する
3. 名前を `BP_Item` にする
4. `BP_Item` をレベルに5個くらい配置する

### うまくいっている目印

- `Outliner` に `BP_Item` が複数ある
- ただの `Cube` ではなく、`BP_Item` を置けている

### なぜこの作業をするの？

**あとで「触れたら消える」「点数が増える」という仕組みを入れるため**です。  
ただの Cube のままだと、毎回同じ設定をやり直すことになります。

### 困ったとき

- 変換前の Cube を増やしてしまった: `BP_Item` を置き直す
- どれが本物かわからない: `Outliner` の名前で確認する

## 3. 触れたら消えて、点数が増えるようにする（35〜60分）

**ここが今日いちばん大事なポイントです。イベントと変数を使って、ゲームのルールを作ります。**

### やること

1. `BP_Item` を開く
2. `Box Collision` を追加し、アイテムに触れたときの判定を作る
3. `OnComponentBeginOverlap (Box)` を追加する
4. `ThirdPersonCharacter` に `Score` 変数を作る
5. プレイヤーが触れたときだけ `Score` を `+1` して、最後に `Destroy Actor` する

### うまくいっている目印

- 触れたアイテムだけ消える
- 2個取るとスコアが2、3個取ると3になる

### なぜこの作業をするの？

**ゲームは「何かが起きたら状態が変わる」の積み重ねで動いています。**  
このステップで、その基本形を体験します。

### 困ったとき

- 反応しない: `Box Collision` の Overlap イベントを使っているか確認する
- 点数が増えない: `Destroy Actor` より前に加算しているか確認する
- 変数がつながらない: `Score` の型が `Integer` になっているか確認する

## 4. ゴールに入ったらクリアにする（60〜80分）

**ここでは、「集めた数が足りているか」を条件で判断します。**

### やること

1. `BP_Goal` を作ってレベルに置く
2. `Box Collision` を追加する
3. `RequiredScore` 変数を作り、値を `5` にする
4. プレイヤーの `Score` を取り出し、`Score >= RequiredScore` で分岐する
5. 条件を満たしたらクリア表示につなぐ

### うまくいっている目印

- 4個以下ではクリアしない
- 5個以上集めてから入るとクリアする

### なぜこの作業をするの？

**「条件を満たしたら次へ進む」という仕組みは、ほとんどのゲームに出てきます。**  
この考え方がわかると、ルールを自分で増やしやすくなります。

### 困ったとき

- 5個で反応しない: 比較演算子が `>` ではなく `>=` になっているか確認する
- ゴールに入っても何も起きない: `Box Collision` の位置と大きさを確認する

## 5. クリア表示を出して、最後まで遊ぶ（80〜90分）

**最後は「完成した」と目でわかる状態にします。**

### やること

1. `Widget Blueprint` を1つ作る
2. テキストに `CLEAR!` と入れる
3. クリア時に `Create Widget -> Add to Viewport` を実行する
4. 最初から最後まで通してプレイする

### 最後に確認すること

- キャラクターを動かせる
- アイテムを取ると数が増える
- 集めてからゴールするとクリアになる
- 自分が作った仕組みを1つ説明できる

## 困ったときの最短チェック

1. `Compile` と `Save` を押す
2. ノードの線が途中で切れていないか見る
3. 変数の型が合っているか見る
4. `Outliner` で正しい Blueprint を置いているか見る

## 講師に伝えるときはこの順

- どのステップで止まったか
- 何を押したか、何をつないだか
- 本当はどうなってほしかったか
- 実際にはどうなったか
