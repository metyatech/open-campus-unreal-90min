---
title: 授業概要
---

この授業は、**オープンキャンパスで「ゲーム開発ってこういうことか」を体験してもらうための90分授業**です。

**成功条件は、受講者がミニゲームを最後まで完成させ、自分が追加した仕組みをひとつ説明できること**です。

## 授業の前提

- 対象: 主に高校既卒以上の初学者
- 想定経験: Unreal Engine 未経験、プログラミング未経験でも参加できる
- 形式: Unreal Engine 5.x / Third Person / Blueprint
- 進め方: 講師の短い実演 → 受講者の操作 → その場で確認

## 90分で作るもの

完成させるのは、**アイテムを集めてゴールするとクリアになるミニゲーム**です。

- Third Person テンプレートでキャラクターを動かす
- アイテムに触れると消える
- アイテムを取るとスコアが増える
- 必要数を集めてゴールするとクリアになる

## なぜ Unreal Engine + Blueprint なのか

- **開始直後から3Dキャラクターを動かせる**
  最初の10分で「ゲームっぽさ」を感じられます。
- **処理の流れを目で追える**
  ノードをつなぐため、当たり判定や条件分岐の考え方が伝わりやすいです。
- **90分授業と相性がよい**
  タイピング時間を減らし、理解と成功体験に時間を使えます。

## 到達目標

- Unreal Engine の基本画面を見て、何をどこで触るか大まかにわかる
- `Score` のような変数を使って、アイテム取得で状態が変わることを体験する
- 条件分岐を使って「一定数集めたらクリア」を作れる
- 自分の言葉で「どの仕組みを作ったか」を説明できる

## 90分の進め方

| 時間 | 何をするか | その時間のゴール |
| --- | --- | --- |
| 0〜15分 | 完成デモ、画面の見方、プロジェクト作成 | まず動く状態にする |
| 15〜35分 | `BP_Item` 作成、レベルに配置 | 自分で触る対象を作る |
| 35〜60分 | Overlap と `Score` 加算 | イベントと変数の意味を体験する |
| 60〜80分 | `BP_Goal` とクリア条件 | 条件分岐でゲームルールを完成させる |
| 80〜90分 | 通しプレイ、振り返り、学科案内 | 成功体験を言葉にして持ち帰る |

## 遅れたときに削る順

- 最初に削る: テキスト装飾や見た目の調整
- 次に削る: アイテム数の増減や配置の作り込み
- 最後まで残す: アイテム回収、スコア加算、ゴールでクリア

**この授業では、豪華さより「遊べる完成」が優先**です。

## この授業で伝えたいこと

- ゲームは見た目だけでなく、**当たり判定・変数・条件分岐**で動いている
- いきなり大作を作るのではなく、**小さい仕組みをつないで完成させる**
- 90分で触れた流れは、今後のゲーム制作やアプリ開発の入口になる
