## 最初に受講者へ伝えること

- 「今日は90分で、ゲームを1本完成させます」
- 「全部を暗記しなくて大丈夫です。まずは見本どおりに完成させましょう」
- 「困ったら、今どのステップで止まっているかを教えてください」

## 90分進行台本

| 時間 | やること | 教員の焦点 | 遅れたときの対応 |
| --- | --- | --- | --- |
| 0〜10分 | 「はじめに」ページを開いてデモ動画を再生 → 「今日の流れ」でタイムライン図を説明 → 全員が UE90min.uproject を開けているか確認 | デモ動画で完成形を見せてから各フェーズの目的を一言ずつ説明する | プロジェクトが開けない受講者は個別対応しつつ、他の受講者はプレイ確認に進む |
| 10〜30分 | BP_Item 作成と配置 | Blueprint 化の意味を一言で伝える | 配置数は3個でもよい |
| 30〜55分 | Overlap と Score 加算 | Destroy Actor より前に加算する順番を徹底する | 見た目の調整は省略 |
| 55〜75分 | BP_Item にクリア判定を追加（Branch / Score >= 5） | Set Score → Branch → Print String「CLEAR!」の接続を全員で確認する | Branch が組み終わらなければ、発展演習を省略して時間を確保する |
| 75〜90分 | 通しプレイ、振り返り | 「自分が作った仕組み」を言葉にさせる | 発表は1〜2名で十分 |

## 詰まったときの声かけ

- 「まずは完璧より、講師画面と同じ状態まで持っていきましょう」
- 「今は意味を全部覚えなくて大丈夫です。つながると何が起きるかを見ましょう」
- 「止まっている人は、今開いている画面がレベルか Blueprint かだけ確認してください」
- 「時間が足りなくなったら、見た目の調整は後回しで大丈夫です」
- 「完成した人は、ゲームを作ろうページの発展演習に進んでください」

## クロージング

「今日やったのは、ゲーム開発のほんの入口です。
でも、当たり判定・変数・条件分岐をつなぐと、ゲームがちゃんと動くことを自分の手で確かめられました。
ここから先は、ルールを1つ増やすたびに、作れるものが広がっていきます。」
