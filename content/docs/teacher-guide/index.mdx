## この授業で扱うこと

この授業では、Unreal Engine 5 の Blueprint を使って、90分で「遊べるミニゲーム」を完成させます。  
高校既卒以上・プログラミング初学者を想定し、操作の成功体験を優先して進めます。

## 授業タイトル・案内文

- タイトル：【特別授業・90分】ゲームプログラミング体験　Unreal Engineで1日ゲーム開発！
- 案内文：プロの現場で使われている「Unreal Engine」を使って、ゲーム制作を体験します。90分の特別授業で、ゲームの仕組みづくりにチャレンジ。ゲームがどう作られているのか、その面白さを実際に触れながら学べます。初めての方でも安心して参加できる内容です！

## 到達目標（授業終了時）

- Third Person テンプレートを起動し、キャラクターを操作できる
- アイテム取得でスコアが増える仕組みを Blueprint で作れる
- ゴール到達でクリア表示が出る最小ゲームループを完成できる

## 教員の進行原則

- 説明は短く、まず動くものを見せる
- 1トピックごとに「実演 3分 → 受講者操作 7分」を基本にする
- 詰まり対応は全体停止より「個別支援 + 先行者課題」で流れを止めない
- C++ は扱わず、Blueprint 操作に集中する

## 90分タイムライン（教員台本）

| 時間    | 進行内容               | 教員アクション                             | 受講者アクション     |
| ------- | ---------------------- | ------------------------------------------ | -------------------- |
| 0-10分  | 導入・今日のゴール共有 | 完成イメージを1分実演。到達目標を3点提示。 | 画面準備、操作確認   |
| 10-20分 | プロジェクト起動       | Third Person（Blueprint）新規作成を実演。  | 同じ手順で作成       |
| 20-35分 | レベル編集基礎         | アイテム（Pickup）配置の流れを実演。       | マップに3〜5個配置   |
| 35-50分 | スコア機能             | 取得時にスコア加算する Blueprint を実演。  | 同じノード構成を作成 |
| 50-65分 | クリア条件             | ゴールトリガー到達で Clear 表示を実演。    | クリア判定を実装     |
| 65-75分 | 調整                   | 取得数・配置・表示文言を調整する例を提示。 | 自分のゲームを調整   |
| 75-85分 | 動作確認・ミニ発表     | 代表2〜3名の画面を投影しコメント。         | プレイ・共有         |
| 85-90分 | まとめ・クロージング   | 学びの言語化、次の学習導線を案内。         | 振り返り             |

## 進行中に使う言い回し（つなぎ文）

- 導入→作業開始：「ここからは私が3分で見本を見せます。終わったら同じ操作を皆さんが7分でやってみましょう。」
- 実演→受講者操作：「今の手順をそのまま再現してください。画面が違う人は手を挙げず、まず隣と見比べて1分だけ自己解決してみましょう。」
- 操作時間終了：「ここでいったん全員ストップ。完成していなくても大丈夫です。次の手順で追いつけるようにします。」
- 詰まりが多い時：「今から共通で詰まりやすい点だけ30秒で直します。その後また個別で回ります。」
- 発表前：「完成度より、どこを工夫したかを1つだけ言えれば十分です。」

## 実演と受講者操作の切り替え基準

- 実演する場面：
  - 初回起動や保存先指定など、間違えると復旧コストが高い操作
  - Blueprint ノード接続の最初の1回
  - クリア条件の分岐など、考え方が重要な箇所
- 受講者に任せる場面：
  - 同じパターンの反復（アイテム追加、文言変更、配置調整）
  - 見た目や難易度の微調整

## 進度差への支援戦略（混在クラス）

- 先行グループ：
  - 追加課題を提示（アイテムを10個に増やす、制限時間を表示する）
  - 役割を依頼（近くの受講者サポート）
- 標準グループ：
  - 台本どおりに完走させる
  - 「いま何分ブロックか」を明示して迷いを減らす
- 詰まりグループ：
  - まず「いま開いている画面」を確認（Content Drawer / Blueprint / Level）
  - エラー解消より先に、動作する最小状態へ戻す
  - 5分以上停止したら、配布済み中間チェックポイントを適用して復帰

## 教員チェックポイント（停止判定）

- 20分時点：全員がプロジェクト起動済み
- 50分時点：半数以上がスコア加算を確認
- 65分時点：半数以上がクリア表示まで到達
- 未達が多数の場合は、演出追加を削って「遊べる完成」を優先

## クロージング用スクリプト（読み上げ例）

「今日は、プロ現場でも使われる Unreal Engine で、ゲームの仕組みを実際に作りました。  
大事なのは、最初から難しいことを全部やることではなく、`動く最小構成` を作ってから広げることです。  
この90分でやった流れは、将来のゲーム制作やアプリ開発にもそのまま使えます。  
興味が出た人は、次は『ルールを1つ追加する』ところから始めてみましょう。お疲れさまでした。」
