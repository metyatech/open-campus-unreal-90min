このページは、**授業を止めないための運営メモ**です。

**最優先は、受講者全員が90分の中で「遊べる状態」まで到達すること**です。  
インストールや個別設定に時間を使いすぎないよう、事前準備とフォールバックを先に決めておきます。

## 結論だけ先に

- **当日に新規インストールや新規アカウント作成をさせない**
- **保存先ルールを最初に固定する**
- **中間チェックポイント用プロジェクトを必ず用意する**
- **複数台で同じ障害が出たら、個別対応より全体案内に切り替える**

## 前日までに終えること

- [ ] 教室PCで Unreal Engine 5.x が起動する
- [ ] `Third Person` / `Blueprint` で新規プロジェクトを作成できる
- [ ] Epic Games Launcher のログイン運用を教員間で統一する
- [ ] 保存先ルールを決めて共有する
- [ ] 中間チェックポイント用プロジェクトを USB と共有フォルダの両方に置く
- [ ] 予備PCと、2人1組に切り替える基準を決めておく

## 当日30分前にやること

1. 教員PCで Unreal Engine を起動する
2. 新規プロジェクトを1つ作る
3. `Play` で移動とジャンプを確認する
4. `BP_Item` の取得と `BP_Goal` のクリア判定を確認する
5. 保存して再起動し、再度開けるか確認する

## 保存場所のルール

**保存先がばらけると、復旧と回収に時間がかかります。**

- 例: `D:\OpenCampus\UE90min\seat-01`
- 席番号や番号札でそろえる
- デスクトップ直下や同期フォルダには保存しない

## Epic Games Launcher 運用

- 授業前にサインインを終えておく
- 授業中にアップデートを始めない
- 受講者ごとの新規アカウント作成はさせない
- ネットワーク制約がある教室では、事前展開済み環境を前提にする

## 授業中の一次対応

| 症状 | まず見ること | その場の対応 |
| --- | --- | --- |
| Launcher にログインできない | 回線、認証情報、プロキシ設定 | 予備PCに切り替えるか、2人1組にする |
| UE の起動が極端に遅い | 空き容量、バックグラウンド更新 | 予備PCへ移動し、問題端末は授業後に確認する |
| プロジェクトを作れない | 保存先権限、パス長、空き容量 | ローカルの短いパスに変えて再作成する |
| `Play` しても操作できない | 入力フォーカス、テンプレート選択 | 画面クリックで入力を有効にし、必要なら作り直す |
| Blueprint で止まった | ノード接続、変数型、参照先 | 中間チェックポイントを適用して復帰させる |
| 画面描画が不安定 | GPU ドライバ、描画設定 | 画質を下げ、厳しければ別端末へ移す |

## フォールバックの優先順位

1. まずは個別支援で復帰させる
2. 5分以上止まるなら中間チェックポイントを配布する
3. 端末問題なら予備PCか2人1組に切り替える
4. 同時多発なら一斉ハンズオンを止め、教員デモ + 追従方式にする

## 授業後にやること

- 成果物を指定フォルダに集約する
- 個人データを削除する
- Launcher のログイン状態を運用ポリシーどおりに戻す
- 起きた障害と復旧方法をメモし、次回のチェックリストに反映する
