# 【特別授業・90分】ゲームプログラミング体験（Unreal Engine / Blueprint）

高卒層の初学者向けに、90分で「動くゲームを完成させる」体験授業パッケージです。  
Unreal Engine 5.x の **Third Person テンプレート**と **Blueprint** を使い、入力より理解を優先して進めます（C++は使いません）。

## この授業で作るもの

第三者視点でキャラクターを動かし、ステージ内のアイテムを集めてゴールするとクリアになるミニゲームを作ります。

- テンプレート：Third Person（Unreal Engine 5.x）
- 実装方法：Blueprint中心
- 完成イメージ：
  - キャラクターを操作して移動・ジャンプできる
  - アイテムを取るとスコアが増える
  - 必要数を集めるとゴール条件を満たせる
  - ゴール到達でクリア表示が出る

## 対象

- 主に高校既卒者（ゲーム制作未経験〜初学者）
- IT・ゲーム分野への進路を検討している人

## 初心者に Unreal + Blueprint が向いている理由

- 3Dキャラクター操作の土台が最初からあり、開始10分で「遊べる状態」まで持っていける
- ノード接続で処理を組むため、タイピング量を抑えつつ「処理の流れ」を可視化できる
- 収集とクリア条件の実装で、ゲームの基本構造（入力・判定・状態変化）を短時間で体験できる

## 学習目標（授業内）

- Unreal Engineの基本画面（レベル、アウトライナ、詳細）を理解する
- Blueprintで「当たり判定」「変数」「分岐」の基礎を体験する
- ゲームの基本ループ（操作→収集→達成）を1本の流れで実装する

## 90分タイムライン（授業運用用）

| 時間     | 内容                                                     | ねらい                         |
| -------- | -------------------------------------------------------- | ------------------------------ |
| 0〜10分  | 導入・完成デモ・到達目標の共有                           | 完成像を先に見せて不安を下げる |
| 10〜20分 | UE5画面の見方と操作（移動・再生・停止）                  | 迷わず作業できる土台を作る     |
| 20〜40分 | Third Personテンプレートでプロジェクト作成、ステージ配置 | 触って動く状態を早く作る       |
| 40〜60分 | アイテム収集（重なったら消える＋スコア加算）             | Blueprintの基本パターンを学ぶ  |
| 60〜75分 | クリア条件（必要数判定＋ゴール到達）                     | 分岐と条件管理を体験する       |
| 75〜85分 | クリア表示の追加と通しプレイ                             | 「完成した」達成感を作る       |
| 85〜90分 | ふりかえり・学科紹介                                     | 次の学びにつなげる             |

## 授業終了時に学生が完成させる状態

- キャラクターを動かして、複数アイテムを回収できる
- 回収数がカウントされる
- 規定数以上でゴール判定が通り、クリア表示が出る
- 学生本人が「どこを作ったか」を1分で説明できる

## 講師向けクイックサマリー（運用メモ）

- 進行の最優先は、60分時点で全員が「回収でスコアが増える」まで到達すること
- 詰まりやすい点は、Overlap接続先の取り違え・変数型の誤り・比較演算子（`>=`推奨）
- 遅延時は UI 演出を最小化し、回収→判定→クリアの主線完成を優先する
