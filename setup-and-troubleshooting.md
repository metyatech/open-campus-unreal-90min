# セットアップ・トラブル対応（教員運用）

## この資料の目的

この資料は、オープンキャンパス90分授業を「止めずに運営する」ための教員向け運用手順です。  
最優先は、全受講者が授業内で遊べる状態まで到達することです。

## 前日チェックリスト（必須）

- [ ] 教室PCで Unreal Engine 5.x が起動し、Third Person（Blueprint）プロジェクトを新規作成できる
- [ ] Epic Games Launcher のログイン方法（共通手順）を教員間で統一済み
- [ ] 演習用の保存先ルール（例：`D:\OpenCampus\UE90min\{学籍番号または席番号}`）を配布済み
- [ ] ネットワーク制限の有無（プロキシ、ポート制限）を情報システム担当に確認済み
- [ ] 当日配布する「中間チェックポイント」プロジェクト一式をUSBと共有フォルダの両方に配置済み
- [ ] 90分版タイムライン（どこを削るかの優先順位）を担当教員全員で確認済み

## 教室PCの前提条件（推奨）

- OS と GPU ドライバが更新済みで、3D表示が安定している
- 各PCで同一の Unreal Engine バージョンを使用する
- 演習保存先はローカル高速ディスクを優先し、ネットワークドライブ直保存は避ける
- 同時起動台数分の空き容量を確保する（更新時の一時領域も見込む）

## Epicアカウント / Launcher 運用方針

- 原則：
  - 当日開始前にサインインを完了しておく
  - 授業中に新規アカウント作成をさせない
- 実務運用：
  - 受講者個別ログイン方式にすると開始が遅れるため、体験授業では教員管理の運用手順を事前に決める
  - Launcher の更新通知が出る場合は、授業中に更新しない
- 公式情報の参照先：
  - UE インストール：`install-unreal-engine`
  - オフライン配布：`offline-installer-of-unreal-engine`
  - 教育機関展開の注意点：`academic-installation`

## 保存場所の運用（事故防止）

- 保存先は最初に全員同じ規則で固定する
- 推奨例：
  - ルート：`D:\OpenCampus\UE90min\`
  - 個別：`D:\OpenCampus\UE90min\seat-01` のように席番号で分離
- 禁止：
  - デスクトップ直下に散在保存
  - OneDrive 等の同期中フォルダへの保存

## ネットワーク / プロキシ注意点

- Launcher 経由配信はネットワーク制約で失敗することがある
- プロキシ環境ではサインインやダウンロードが不安定になる場合があるため、事前に疎通確認する
- 教室ネットワークが不安定な場合：
  - 当日の新規ダウンロードを避ける
  - 事前展開済み環境とオフライン手段を優先する

## 当日スモークテスト（授業開始30分前）

1. 教員PCで Unreal Engine を起動する
2. Games > Third Person > Blueprint で新規プロジェクトを作る
3. Play でキャラクター移動とジャンプを確認する
4. アイテム1個を置き、取得イベントが発火するか確認する
5. ゴールトリガーで Clear 表示が出るか確認する
6. 保存して再起動し、プロジェクト再オープンできるか確認する

## よくある障害と対処

| 症状                           | 一次切り分け                        | 即時対処                             |
| ------------------------------ | ----------------------------------- | ------------------------------------ |
| Launcher にログインできない    | 回線 / 認証情報 / プロキシ設定      | 予備PCに切替。授業中の新規登録は中止 |
| UE 起動が極端に遅い            | ディスク空き / バックグラウンド更新 | 不要プロセス停止、予備PCへ移動       |
| プロジェクト作成失敗           | 保存先権限 / パス長 / 空き容量      | ローカル短パスに変更して再作成       |
| Play で操作不能                | 入力フォーカス / テンプレート差異   | Third Person テンプレート再作成      |
| Blueprint 接続エラーで進行停止 | 参照切れ / ノード接続ミス           | 中間チェックポイントを適用して復帰   |
| 画面描画が乱れる               | GPU ドライバ / 描画設定過負荷       | 画質を下げる、別端末へ切替           |

## フォールバック計画（失敗時の授業継続）

- インストール失敗時：
  - 事前配布済みプロジェクトを配り、「配置調整」と「数値変更」中心で体験を継続
- アカウント失敗時：
  - ログイン不要で起動済みの予備PCへ案内し、2人1組で進行
- ネットワーク障害時：
  - オンライン依存操作を停止し、ローカル保存とオフライン教材で継続
- 複数台同時障害時：
  - 一斉ハンズオンを中断し、教員デモ + 指示書追従方式に切り替え

## 授業後のクリーンアップ

- 受講者成果物を回収（指定フォルダに集約）
- ローカル保存領域の個人データを削除
- Launcher のログイン状態を教室運用ポリシーに従って解除
- 障害ログを記録（発生時刻、台数、復旧方法、次回予防策）
- 次回向けに「前日チェックリスト」の改善点を1つ以上追加

## 公式ドキュメントとの整合メモ（教員向け）

- UE本体の導入手順は Epic 公式の UE 5.6 インストール手順に合わせる
- 教室配備やネットワーク制約下の運用は、オフラインインストーラーおよび教育機関向け展開資料の考え方を優先する
- 実際の運用可否は校内ネットワークポリシーが最優先のため、事前検証結果を授業計画に反映する
